# Contribution Guide

**Audience**: Developers contributing to Autonomey

## Purpose

How to contribute to Autonomey development.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a branch for your changes
4. Make your changes
5. Test your changes
6. Submit a pull request

## Development Setup

See [Quick Start](../deployment/quick-start.md) for setup instructions.

## Code Style

See [Code Style](code-style.md) for detailed standards.

## Areas for Contribution

### Smart Contracts

- Security improvements
- Gas optimization
- New features
- Bug fixes

### Frontend

- UI/UX improvements
- New components
- Bug fixes
- Performance optimization

### Documentation

- Clarifications
- Examples
- Tutorials
- Translations

### Testing

- Test coverage
- Integration tests
- End-to-end tests

## Pull Request Process

1. **Create Feature Branch**
   - Branch from `main`
   - Use descriptive branch names

2. **Make Changes**
   - Follow code style
   - Write tests
   - Update documentation

3. **Test Changes**
   - Run all tests
   - Test on testnet
   - Verify no regressions

4. **Submit PR**
   - Clear description
   - Link to issues
   - Request review

## Code Review

- All PRs require review
- Address feedback promptly
- Keep PRs focused and small

## Reporting Issues

- Use GitHub Issues
- Provide clear description
- Include reproduction steps
- Add relevant labels

## Questions?

- Open a GitHub Discussion
- Check existing documentation
- Review closed issues
